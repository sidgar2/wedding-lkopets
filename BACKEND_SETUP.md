# Налаштування бекенду

Google Sheets + Apps Script для RSVP-форми весільного сайту.

---

## Крок 1 — Створити Google Таблицю

1. Відкрийте [sheets.new](https://sheets.new)
2. Назвіть таблицю: **Wedding RSVP — Любомир та Маряна**
3. Скопіюйте **ID таблиці** з URL — це частина між `/d/` і `/edit`:
   ```
   https://docs.google.com/spreadsheets/d/  ← ось це →  /edit
   ```

---

## Крок 2 — Налаштувати вкладки

### Вкладка `Guests` (заповнюється вручну)

Перша вкладка має називатись **Guests**. Перший рядок — заголовки:

| A | B | C | D | E |
|---|---|---|---|---|
| code | name | maxGuests | side | notes |

Приклад даних (рядки 2, 3, ...):

| k7m2pq | Олег та Світлана Коваль | 2 | groom | |
| a3bx9r | Родина Петренків | 4 | bride | 2 дітей |

> Коди та дані генеруються скриптом `scripts/generate-codes.ts` → файл `guests-import.tsv`.  
> Відкрийте TSV у текстовому редакторі, виділіть усе, вставте у клітинку A1 таблиці.

### Вкладка `RSVP` (заповнюється автоматично)

Друга вкладка має називатись **RSVP**. Перший рядок — заголовки:

| A | B | C | D | E | F |
|---|---|---|---|---|---|
| timestamp | guestCode | name | guestsCount | attendance | comment |

Більше нічого не треба — скрипт дописуватиме рядки автоматично.

---

## Крок 3 — Apps Script

1. У Google Таблиці відкрийте **Extensions → Apps Script**
2. Видаліть весь вміст редактора за замовчуванням
3. Скопіюйте вміст файлу `apps-script/Code.gs` і вставте
4. Замініть рядок:
   ```js
   var SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID_HERE';
   ```
   на реальний ID з Кроку 1
5. Збережіть проєкт (Ctrl+S або кнопка 💾)

---

## Крок 4 — Деплой Web App

1. Натисніть **Deploy → New Deployment**
2. Тип: **Web App**
3. Налаштування:
   - **Execute as:** Me *(від вашого імені)*
   - **Who has access:** Anyone *(потрібно, щоб сайт міг зверталися до скрипту)*
4. Натисніть **Deploy**
5. При першому деплої Google попросить підтвердити дозволи — погодьтесь (це стандартна процедура для доступу до таблиці)
6. Скопіюйте **Web App URL** — він матиме вигляд:
   ```
   https://script.google.com/macros/s/AKfycb.../exec
   ```

---

## Крок 5 — Додати URL до проєкту

1. Скопіюйте файл `.env.example` → `.env`:
   ```bash
   cp .env.example .env
   ```
2. Вставте URL зі Кроку 4:
   ```
   VITE_RSVP_ENDPOINT=https://script.google.com/macros/s/AKfycb.../exec
   ```
3. Перезапустіть dev-сервер: `npm run dev`

> **Важливо:** Файл `.env` вже додано до `.gitignore` — він не потрапить у репозиторій.

---

## Крок 6 — Тест

Відкрийте сайт і заповніть форму. Перевірте:
- Відповідь з'явилась у вкладці **RSVP** таблиці
- Якщо налаштований Telegram — прийшло сповіщення

Якщо форма показує помилку — відкрийте DevTools → Network і перевірте відповідь від `script.google.com`.

---

## Генерація кодів для гостей

```bash
# Відредагуйте масив GUESTS у scripts/generate-codes.ts
# Запустіть:
npx tsx scripts/generate-codes.ts
```

Скрипт виведе посилання у консоль і збереже `guests-import.tsv`.  
Вставте TSV у вкладку **Guests** таблиці (A1, вставити як текст).  
Розішліть посилання гостям — кожне унікальне.

## Опціонально: Telegram-сповіщення

Якщо хочете отримувати миттєві сповіщення про кожен RSVP:

1. **Створіть бота:** Напишіть [@BotFather](https://t.me/BotFather) команду `/newbot`, дайте ім'я → отримаєте `token`
2. **Напишіть своєму боту** будь-яке повідомлення (щоб активувати чат)
3. **Дізнайтесь chat_id:** Відкрийте в браузері:
   ```
   https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
   ```
   Знайдіть `"chat":{"id": 123456789}` — це і є ваш `chat_id`
4. **Додайте в Apps Script** перед деплоєм:
   ```js
   var TELEGRAM_BOT_TOKEN = '1234567890:AAF...';
   var TELEGRAM_CHAT_ID   = '123456789';
   ```
5. Зробіть новий деплой (**Deploy → Manage Deployments → Edit → нова версія**)

> Цей крок **необов'язковий** — без нього все працює, просто не буде миттєвих сповіщень. Відповіді гостей завжди видно у вкладці RSVP таблиці.
